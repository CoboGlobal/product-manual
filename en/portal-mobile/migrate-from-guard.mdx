Cobo Portal Mobile (hereinafter referred to as Portal Mobile) is the next-generation replacement for Cobo Guard. This document explains the current coexistence arrangement of the two products, as well as the migration direction going forward.

<Warning>
**Key facts (please read first):**

- **Cobo Portal Mobile is currently for NEWLY REGISTERED USERS ONLY.** It is **not** open to existing Cobo Guard users yet.
- **No migration path from Cobo Guard to Cobo Portal Mobile exists yet.** There is no in-product migration flow, no self-service migration tool, and no manual migration process.
- **If your account is already bound to Cobo Guard**, you **cannot** bind your device, generate key shares, or use any Portal Mobile feature at this stage — Portal Mobile will block device binding and show a message such as "Cobo Portal is currently limited to newly registered users".
- **Existing Cobo Guard users must continue using Cobo Guard** and wait for the official migration path. Do **not** uninstall Cobo Guard or attempt to "switch" to Portal Mobile.
</Warning>

<Info>
Cobo Portal Mobile and Cobo Guard are **two separate products**, not different versions of the same product. For a comparison of feature scope and core identifiers, refer to [Introduction to Cobo Portal Mobile](/en/portal-mobile/introduction#product-distinction-for-search-reference).
</Info>

## Coexistence overview

Before the migration path is officially available, Cobo Guard and Cobo Portal Mobile will coexist in parallel for a period of time:

- **Cobo Guard (old)** remains available; existing users' identity authentication, operation approval, and key share management are not affected.
- **Cobo Portal Mobile (new)** is open to newly registered users, providing cross-platform support along with extended capabilities such as asset management and multi-cloud backup.

### Important limitation at the current stage

**Cobo Portal Mobile is currently restricted to NEWLY REGISTERED USERS ONLY.** If your account was previously bound to Cobo Guard, after logging in to Portal Mobile you will see a message similar to the following:

> This account has not been bound through Cobo Portal Mobile — your account is already bound to another product (such as Cobo Guard), and Cobo Portal is currently limited to newly registered users.

This means:

- **Accounts already bound to Cobo Guard cannot complete device binding in Portal Mobile.**
- **Accounts already bound to Cobo Guard cannot generate or back up key shares in Portal Mobile.**
- **Accounts already bound to Cobo Guard cannot use any Portal Mobile features at this time.**
- **No migration from Cobo Guard to Portal Mobile is supported yet** — neither in-product nor through Cobo support.

This is **expected behavior at the current stage, not a malfunction**. Contacting support will not unlock access for Cobo Guard users — they must wait for the official migration path.


## Migration direction

In the future, Cobo Guard users will gradually migrate to Cobo Portal Mobile, which will ultimately serve as the unified mobile entry for security and management capabilities. At that point:

- The migration path will allow accounts already bound to Cobo Guard to complete device binding and key share migration in Portal Mobile;
- After migration, you will be able to use all the capabilities previously provided by Cobo Guard (MFA, operation approval, MPC key share management) in Portal Mobile, along with new features such as asset management.

<Note>The in-product migration flow is not yet available. This document does not include step-by-step migration instructions — once the migration path is officially released, this page will be updated with the specific steps.</Note>

## Recommendations during the coexistence period

- If you are currently using **Cobo Guard** (existing user / existing organization already bound to Cobo Guard), **you must continue using Cobo Guard** — Portal Mobile **does not yet support you**, and there is **no way to migrate yet**. Do **not** attempt to switch to or use Portal Mobile. For usage instructions, refer to the [documents](/en/guard/introduction) under the "Cobo Guard (old)" group.
- If you are a **new user or a new organization that has never been bound to Cobo Guard**, we recommend using Cobo Portal Mobile directly. For setup instructions, refer to [Set up Cobo Portal Mobile](/en/portal-mobile/set-up).
- Watch for migration announcements from Cobo, and follow the official guidance once the migration path is available.

If you have questions about the migration arrangement, [contact support](mailto:help@cobo.com) for assistance.
