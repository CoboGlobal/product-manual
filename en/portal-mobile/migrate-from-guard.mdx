The Cobo Portal Mobile App (hereinafter referred to as Portal Mobile) is the next-generation replacement for Cobo Guard. This document explains the current coexistence arrangement of the two products, as well as the migration direction going forward.

<Info>
The Cobo Portal Mobile App and Cobo Guard are **two separate products**, not different versions of the same app. For a comparison of feature scope and core identifiers, refer to [Introduction to Cobo Portal Mobile App](/en/portal-mobile/introduction#product-distinction-for-search-reference).
</Info>

## Coexistence overview

Before the migration path is officially available, Cobo Guard and the Cobo Portal Mobile App will coexist in parallel for a period of time:

- **Cobo Guard (old)** remains available; existing users' identity authentication, operation approval, and key share management are not affected.
- **Cobo Portal Mobile App (new)** is open to newly registered users, providing cross-platform support along with extended capabilities such as asset management, multi-cloud backup, and an AI assistant.

### Important limitation at the current stage

Currently, the Cobo Portal Mobile App is primarily intended for **newly registered users**. If your account was previously bound to Cobo Guard, after logging in to Portal Mobile you will see a message similar to the following:

> This account has not been bound through the Cobo Portal mobile app — your account is already bound to another app (such as Cobo Guard), and Cobo Portal is currently limited to newly registered users.

This means that **accounts already bound to Cobo Guard cannot yet complete device binding in Portal Mobile**. This is expected behavior at the current stage, not a malfunction.

## Migration direction

In the future, Cobo Guard users will gradually migrate to the Cobo Portal Mobile App, which will ultimately serve as the unified mobile app for security and management capabilities. At that point:

- The migration path will allow accounts already bound to Cobo Guard to complete device binding and key share migration in Portal Mobile;
- After migration, you will be able to use all the capabilities previously provided by Cobo Guard (MFA, operation approval, MPC key share management) in Portal Mobile, along with new features such as asset management and the AI assistant.

<Note>The in-app migration flow is not yet available. This document does not include step-by-step migration instructions — once the migration path is officially released, this page will be updated with the specific steps.</Note>

## Recommendations during the coexistence period

- If you are currently using **Cobo Guard**, continue using Cobo Guard — there is no need to switch immediately. For usage instructions, refer to the [documents](/en/guard/introduction) under the "Cobo Guard (old)" group.
- If you are a **new user or a new organization**, we recommend using the Cobo Portal Mobile App directly. For setup instructions, refer to [Set up Cobo Portal Mobile App](/en/portal-mobile/set-up).
- Watch for migration announcements from Cobo, and follow the official guidance once the migration path is available.

If you have questions about the migration arrangement, [contact support](mailto:help@cobo.com) for assistance.
