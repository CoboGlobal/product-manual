如果您正在更换新设备、无法再访问旧设备，或重新安装了 Cobo Portal Mobile App（以下简称 Portal Mobile），则需要在新设备上重新完成设备绑定，以确保 App 正常使用。

<Note>为避免潜在风险，我们不建议卸载并重新安装 Portal Mobile。卸载会清除本机保存的私钥分片。如果您无意间卸载并重装了 App，也需要执行本文档中的重置流程。在这种情况下，「旧设备」与「新设备」指同一台设备。</Note>

请根据您是否创建过私钥分片，阅读适用的部分：

- [适用于已创建私钥分片的用户](#key-share-holders)
- [适用于尚未创建私钥分片的用户](#non-key-share-holders)

## 适用于已创建私钥分片的用户 <a id="key-share-holders"></a>

如果您的设备作为私钥分片持有者参与 MPC 钱包密钥管理，请按以下步骤在新设备上重置。

### 第 1 步：在新设备上登录

1. 在新设备上[下载 Cobo Portal Mobile App](/cn/portal-mobile/download)。
2. 打开 App，完成设备锁屏与生物识别设置。
3. 使用与原设备相同的账户登录。

### 第 2 步：选择如何继续

Portal Mobile 检测到该账户此前已绑定其他设备时，会显示**检测到新设备**页面，您可以选择：

- **仅查看**：仅可浏览余额与历史记录，无法在本机签名或审批。原设备保留完整权限。
- **切换至本设备**：将本机设为受信任的签名设备。完成 MFA 验证后，原设备将失去访问权限。

    <Warning>出于安全考虑，切换设备后**发送功能将暂停 24 小时**，期间查看资产与处理审批仍可正常使用。</Warning>

如需在新设备上继续签名与审批，请选择**切换至本设备**，然后点击**继续**。

### 第 3 步：恢复私钥分片

选择**切换至本设备**后，Portal Mobile 会进入**私钥分片恢复**流程，并询问您的备份情况：

- **我有备份文件**：备份文件可在本设备访问（iCloud、Google Drive 或本地保存），且您记得创建备份时设置的恢复口令。请按下方[恢复私钥分片的步骤](#restore-steps)操作。
- **我的备份文件已丢失**：请联系组织管理员在 Cobo Portal 网页版上发起重新分发，由 Vault 中的其他密钥组为您重新生成私钥分片；随后在本设备上完成私钥分片生成任务。

#### 恢复私钥分片的步骤 <a id="restore-steps"></a>

1. 在**恢复私钥分片**页面，选择您的备份文件存储位置：**iCloud**、**Google Drive** 或**本地文件**。

2. 选择需要恢复的私钥分片对应的 **TSS Node ID**（请确保与 Cobo Portal 中显示的 TSS Node ID 一致），然后点击**继续**。

    <Note>如果所选 TSS Node ID 与当前账户不一致，请重新选择正确的备份文件。</Note>

3. 输入创建备份时设置的**恢复口令**，点击**确定**。

4. 恢复完成后，Portal Mobile 会显示**私钥分片恢复成功**。

至此，您已完成 Portal Mobile 的重置流程。如果您选择了**切换至本设备**，请留意发送功能的 24 小时冻结提示。

### 关于换绑后的 24 小时冻结

完成设备切换后，为保护账户安全，相关访问会被冻结 24 小时。在此期间，您可能会看到「因换绑，账户暂时无法使用」的提示。冻结结束后即可正常使用全部功能。如有疑问，请[联系客服](mailto:help@cobo.com)。

## 适用于尚未创建私钥分片的用户 <a id="non-key-share-holders"></a>

如果您从未创建过私钥分片，重置流程更为简单：

1. 在新设备上[下载 Cobo Portal Mobile App](/cn/portal-mobile/download)，完成设备锁屏与生物识别设置。
2. 使用原账户登录。
3. 按提示[重新绑定本设备](/cn/portal-mobile/set-up#bind-device)。
4. 如果 App 检测到此前绑定过其他设备，请在**检测到新设备**页面选择**切换至本设备**，并完成 MFA 验证。

完成后，您即可在新设备上正常使用 Cobo Portal Mobile App。

<Note>如果在重置过程中遇到问题，或无法完成身份验证，请[联系客服](mailto:help@cobo.com)获取帮助。</Note>
