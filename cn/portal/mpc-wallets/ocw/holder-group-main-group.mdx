## 主控私钥
在创建 MPC 钱包之前，必须先创建主控私钥。成功创建主控私钥之后，便可以开始使用 MPC 钱包。

为了最大程度确保资产的安全性，最好尽快创建签名私钥和恢复私钥，并备份所有私钥。可以创建多个签名私钥来满足业务需求，但建议只创建一个恢复私钥。

<a id="create-main-group"></a>
### 创建主控私钥
为 MPC Vault 创建主控私钥时，可以选择移动端签名人或服务端签名人。在继续操作之前，请先了解下表中每种签名人的主要用途和使用前提。

<a id="signer-type"></a>
<table class="six-columns">
  <tr>
    <th>签名人类型</th>
    <th>移动端签名人</th>
    <th>服务端签名人</th>
  </tr>
  <tr>
    <td>主要用途</td>
    <td>
      移动端签名人是一名团队成员，负责通过 [Cobo Guard](/cn/guard/introduction) 安全签署交易。其 TSS Node ID 由 Cobo Guard 生成，并将自动填入。
    </td>
    <td>
      服务端签名人是一个服务器节点，用于自动通过 TSS Node 软件包签署交易。其 TSS Node ID 通过 Cobo 提供的 TSS 软件包生成，且必须在设置钱包之前生成。有关进一步说明，请参见[服务端签名人介绍](https://www.cobo.com/developers/v2_cn/guides/mpc-wallets/server-co-signer/tss-node-overview)。<Info>对于高频交易，我们建议选择服务端签名人进行自动签名。</Info>
    </td>
  </tr>
  <tr>
    <td>前提步骤</td>
    <td>
      您选择的私钥分片持有者需要完成以下操作：
      <ol>
        <li><a href="/cn/guard/download">下载 Cobo Guard</a>。</li>
        <li><a href="/cn/accounts/set-up-mfa#set-up-cobo-guard">将 Cobo Guard 设置为 Cobo Accounts 多重身份验证 (MFA) 方法之一</a>。</li>
        <li>被<a href="/cn/portal/organization/manage-members">添加为您的团队成员</a>。</li>
      </ol>
    </td>
    <td>
      您选择的私钥分片持有者需要向您提供他们生成的 TSS Node ID。
    </td>
  </tr>

决定好签名人并完成前提步骤后，请按照以下说明创建主控私钥。
<Note>只有拥有操作员，经理，管理员[角色](/cn/portal/organization/roles-and-permissions)的用户能够创建主控私钥。</Note>

1. 登录 [Cobo Portal](https://portal.cobo.com/login)。
2. 点击 <img src="/cn/images/mpc-wallet/wallets-icon.svg" className="icon"></img>> **MPC 钱包**。
3. 选择 Vault，然后点击 Vault 名称下的**创建主控私钥**。
<img src="/cn/images/mpc-wallet/create-main-key-group.png" className="screenshot_full_screen"/>
4. 在**创建主控私钥**对话框中，配置**私钥分片持有者2**：<a id="on-the-create-main-group-dialog"></a>
    - 要使用**移动端签名人**，选择一个**持有者**（**TSS Node ID** 将自动填入），持有者可以是任意一名团队成员，其用户角色不限。然后点击**确认**。

      <img src="/cn/images/mpc-wallet/mobile-co-signer.png" className="screenshot_modal"/>

    - 要使用**服务端签名人**，输入名称，填写使用 TSS Node 软件包生成的 **TSS Node ID**，然后点击**确认**。有关进一步说明，请参见[服务端签名人介绍](https://www.cobo.com/developers/v2_cn/guides/mpc-wallets/server-co-signer/tss-node-overview)。
    
5. 请通知相关的私钥分片持有者进行确认。
    - **注意：**
        - 如果使用移动端签名人，选定的私钥分片持有者需要使用 Cobo Guard 生成私钥分片，详情参见[生成私钥分片](/cn/guard/generate-and-back-up-key-shares#生成私钥分片)。
        - 如果使用服务端签名人，请确保在 24 小时内让 TSS Node 上线。TSS Node 上线后，确认流程完成。

7. 私钥分片生成过程于第 5 步完成后，主控私钥分片持有者的**状态**将显示为**活跃**。
<Info>您可能需要刷新 Cobo Portal 页面以查看最新结果。</Info>
<Info>注意：私钥成功生成后，将自动为该 Vault 创建一个默认钱包。</Info>
