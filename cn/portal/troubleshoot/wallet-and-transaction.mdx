## 交易状态

### 交易失败或者一直卡在某个状态怎么办？

**适用范围：所有钱包**

您可以在 Cobo Portal > **钱包** > **交易记录**中查看交易状态及所需操作。如仍需帮助，请联系 Cobo 客服，联系方式：help@cobo.com。

### 充币为什么还没到账？

**适用范围：所有钱包**

如果您发起充币后长时间未到账，建议按以下顺序排查：

1. 在区块链浏览器中查询交易状态
    - 确认交易已成功上链（Success）。
    - 确认已达到所需确认数（Confirmations）。

2. 核对充币信息是否正确
    - 充币地址是否正确。
    - 充币链/网络是否正确。
    - 若该链或资产要求填写 Tag/Memo/备注，请确认填写无误。

3. 检查是否存在入账延迟的常见原因
    - 网络拥堵导致确认时间变长。
    - 部分 Layer2 网络（如 Arbitrum、Optimism、Base、Zora 等）需要等待 L1 最终确认（Finalized）后才会入账：即使您在 L2 浏览器中看到交易成功，也可能需要等待该交易在 L1 达到 Finalized 状态后，平台侧才会完成入账。
    - 系统维护或服务异常。

如需查询 Cobo Portal 对不同链的入账确认数要求，请参阅[支持的代币与链](/cn/portal/supported-tokens-and-chains#代币入账确认数)。

如果问题仍然存在，请准备交易哈希（TxHash）、充币地址、币种与链信息，并联系 Cobo 客服（help@cobo.com）。

## 资产与余额

### 为什么看不到某个代币/链？代币和链不受支持怎么办？

**适用范围：所有钱包**

您购买的 Cobo 全托管钱包订阅包所支持的代币和链全部显示在钱包界面。您也可以在[支持的代币与链](/cn/portal/supported-tokens-and-chains)中查看 Cobo Portal 当前支持的代币与链列表。如果您购买的订阅包中不包括或者目前 Cobo 尚不支持您所需的代币或链，敬请联系 Cobo 客服寻求帮助，联系方式：[help@cobo.com](mailto:help@cobo.com)。

### 转入了不支持的 EVM 链资产，怎么转出？

**适用范围：全托管钱包（Web3 钱包）、MPC 钱包**

如果您不小心把某个不受 Cobo Portal 支持的 EVM 代币转入到了您的地址上，将无法通过 Cobo Portal 的转账流程直接转出。此时您可以通过合约调用的方式从链上转出该资产。可用方式如下：

1. 通过调用智能合约接口转出（API）
    - 适用于：通过程序化方式自行转出。您需要自行构造合约调用所需的 calldata。
    - 参考文档：[Call smart contract](https://www.cobo.com/developers/v2/api-references/transactions/call-smart-contract)。

2. 通过 Cobo Connect 在区块链浏览器上操作合约转出
    - 适用于：在浏览器中通过合约交互界面（例如 Etherscan 的合约交互）发起合约调用。
    - 参考文档：[设置 Cobo Connect](/cn/cobo-connect/set-up)。

    <Warning>合约调用存在资产损失风险（例如网络选错、合约地址选错、数量单位填写错误、目标地址不可逆等）。建议先小额测试。</Warning>

    操作步骤（以 Etherscan 为例）：

    1. 打开区块链浏览器上的代币页面（Token 页面）。
    2. 进入 **Contract** 标签页并打开 **Write Contract**。
    3. 点击 **Connect to Web3**，选择 **Cobo Connect**，并在 Cobo Connect 中选择要操作的 Cobo Portal 钱包地址完成连接。
    4. 找到 `transfer` 方法（通常是 `transfer(address to, uint256 amount)`），填写接收地址与数量（按最小单位填写）。
    5. 点击 **Write** 提交，并按流程在 Cobo Guard 中完成确认/审批。

    请确保您已完成以下准备：

    1. 选择支持该代币的 EVM 网络，并准备足够的原生 Gas（例如 ETH）支付手续费。
    2. 若使用 Cobo Connect，请先将对应的钱包地址导入到 Cobo Connect。

如仍无法处理，请联系 Cobo 客服（help@cobo.com）。

### 全托管钱包（资产钱包）如何进行自动代币归集？

**适用范围：全托管钱包（资产钱包）**

全托管钱包（资产钱包）的自动代币归集由 Cobo 系统内部自动实现，用于将充币地址的资金自动归集到指定地址/钱包中。您无需手动开启开关，也无需日常管理归集流程。

### 自动代币归集什么时候触发？

**适用范围：全托管钱包（Web3 钱包）、MPC 钱包**

对于启用了自动代币归集的全托管钱包（Web3 钱包）与 MPC 钱包，自动代币归集的触发时机和执行策略由 Cobo 系统根据内部规则自动管理。如果您发现资金入账到充币地址后一段时间仍未归集，请先确认该地址在链上确实已收到资金，并注意归集可能会受到网络拥堵、手续费波动、以及平台侧策略执行节奏等因素影响。

参考文档：[自动代币归集概览](/cn/portal/auto-sweep-overview)。

### 为什么充币地址上有些币种无法完全归集？

**适用范围：全托管钱包（Web3 钱包）、MPC 钱包**

在某些区块链网络上，进行自动代币归集操作后，充币地址上仍然可能剩下一小部分资金。主要原因如下：

- 部分币种需要保留最低余额：某些区块链（如 XRP）要求地址必须保留一定的最低余额。例如，XRP 地址至少需要保留 1 XRP，所以即使归集后，充币地址上也会剩下 1 XRP，无法转走。
- L2 自动代币归集中的 L1 费用估算误差：在 L2 网络（如 Base）上交易时，自动代币归集交易需要支付 L1 费用，但实际消耗的费用可能低于预估值。自动代币归集完成后，充币地址可能会剩余 `区块链预估的 L1 费用 - 实际支付的 L1 费用`的金额，这部分无法完全归集。
- EVM 交易的 Gas 估算误差：在 EVM 兼容链（如 Ethereum、BSC）上，自动代币归集交易需要支付 Gas 费，Gas 费是基于预估的最大消耗量（Gas Limit）计算的。但实际执行交易时，消耗的 Gas 可能低于预估值，导致 `Gas Price * 未消耗的 Gas` 的金额仍留在充币地址上。

## 账号与权限

### 无法创建钱包怎么办？

**适用范围：所有钱包**

如果在创建新钱包时遇到困难，请考虑以下可能性及相应解决方案：

1. 用户权限受限
    - 问题：您可能没有创建新钱包所需的权限。
    - 解决方案：确保您具有[所需的权限](/cn/portal/organization/roles-and-permissions)，或联系您的管理员。

2. 网络问题/服务器停机
    - 问题：您的网络可能不稳定，或者 Cobo 服务器正在维护中或暂时不可用。
    - 解决方案：确保您的网络连接稳定，且 Cobo 服务器未处于维护状态。点击[此处](https://status.cobo.com/)查看 Cobo 的系统状态。

## 连接与集成

### 无法访问已连接的交易所账户怎么办？

**适用范围：交易所钱包**

您连接的交易所账户中出现 **“连接失败”**、**“未找到 API”** 或类似的错误消息，可能由多种原因引起，如 API Key 过期或被删除、API 权限不足、交易所服务器上的网络问题、交易所账户被冻结或受限、KYC 问题等。

若要解决此类问题，请点击 <img src="/cn/images/exchange-wallet/24.png" className="icon"/> 重新建立连接。如果问题仍然存在，请检查您的交易所设置，并确认网络连接正常、API KEY 有效、拥有必要权限且是最新的，同时您的账户处于活跃状态。

参考文档：[设置交易所钱包](/cn/portal/exchange-wallets/set-up)

### 无法连接交易所账户怎么办？

**适用范围：交易所钱包**

如果您无法连接交易所账户，请确保先完成以下步骤，然后重试：

1.  在交易所 API 设置中将 Cobo IP (`18.177.234.7`及`18.180.16.190`) 列入白名单，并确保正确输入 API Key 类型和权限。
2.  确保 API Key、API Secret 和口令正确无误。
3.  确保您的交易所账户处于活跃状态，并且没有 KYC 待处理或其他问题。
4.  验证您的互联网连接是否稳定。
5.  确认交易所的系统或服务没有因某种原因无法正常运行、无法提供服务，或正在维护。

参考文档：[设置交易所钱包](/cn/portal/exchange-wallets/set-up)
