## A
### Address Book（地址簿）
地址簿是一个简单易用的提现地址管理器，类似于手机上的联系人列表。它作为一个综合性目录，用于存储、团队和访问您已知的提现地址。地址簿简化了处理冗长字母数字地址的复杂过程，使交易更加高效和安全。

### API Key
API Key 是密钥对中的公钥，用于识别发起 API 请求的客户端。API Key 与权限和钱包范围相关联，这些权限和范围定义了客户端可以执行的操作以及可以访问的钱包。不同的 API Key 可以分配不同的权限和钱包范围，从而实现对客户端功能的精细化控制。

### Approval Quorum（审批门槛）
审批门槛是指在 Cobo Portal 中授权交易或风控规则变更所需的最少审批人数。这有助于通过确保关键操作需要多方授权来维护安全性。

### Asset Wallets（资产钱包）
资产钱包是全托管钱包的一种子类型，允许您转移代币（发起充值或提现）。您可以根据业务需求创建多个钱包。

## C
### Cobo Connect
Cobo Connect 是一个 Chrome 扩展程序，使您能够将 Cobo Portal 钱包与 dApp 连接。建立连接后，Cobo Connect 将识别来自 dApp 的每个交易和消息，并根据您在 Cobo Portal 中预先配置的风控规则处理它们。

### Cobo Guard
Cobo Guard 是一款多功能安全 iOS App，提升您在移动端的数字资产安全性和保护。它作为 MPC 密钥分片管理器和多重身份验证（MFA）方式，用于审批登录和交易等操作。

### Cobo Portal
Cobo Portal 是一个一站式平台，为您提供安全高效的数字资产管理方式。凭借其经过实战检验的安全架构和对代币及链的广泛支持，Cobo Portal 提供多种加密钱包以满足您的特定需求。此外，它还提供了广泛的功能，包括全面的风控措施、无缝访问 Web3/DeFi 网络、直观的数据概览，以及丰富的 Cobo 和第三方 App 选择。

### Cobo Safe
Cobo Safe 是基于 [Safe Module](https://docs.safe.global/advanced/smart-account-modules) 构建的智能合约。Safe Module 扩展了 Safe&#123;Wallet&#125; 的功能，同时将模块逻辑与 Safe&#123;Wallet&#125; 的核心合约分离。作为插件集成在 Safe&#123;Wallet&#125; 中后，Cobo Safe 实现了基于用户角色的精细化链上访问控制。您可以授权外部账户（EOA）通过单签执行由多个 Safe&#123;Wallet&#125; 签名人预授权的操作。

### Custodial Wallets（全托管钱包）
全托管钱包是一种由 Cobo 负责保管私钥的钱包类型，为用户提供机构级别的安全性和便捷的资产管理服务。Cobo 提供资产钱包和 Web3 钱包两种全托管钱包。

## D
### Delegate Key
Delegate Key 用于代表 Safe&#123;Wallet&#125; 签名人发起（只发起不执行）多签交易。它是由 Cobo 创建的地址，在 Delegate Key 设置过程中自动分配给您。

### Delegate
Delegate 可以使用单签执行由 Safe&#123;Wallet&#125; 签名人授权的操作。此授权是您在 Cobo Portal 上为智能合约钱包（Safe&#123;Wallet&#125;）设置链上交易规则时授予的。Delegate 可以是来自 MPC 钱包（机构钱包）或 Cobo Portal 外部的外部账户（EOA）的地址。

## E
### Exchange Wallets（交易所钱包）
该钱包类型将您的多个交易所账户整合到一个统一界面中，让您能够便捷地查看和管理各交易所资产，并支持在交易所账户与 Cobo Portal 其他钱包之间进行无缝资金划转。

### Externally Owned Accounts, EOA（外部账户）
由私钥控制的标准以太坊账户，可以发起交易。与智能合约钱包不同，外部账户不能执行自定义逻辑或实现高级安全功能。

## G
### Gas fees（Gas 费）
在区块链网络上执行交易所需的成本。

### Gas limits（Gas 限制）
您愿意为在以太坊虚拟机（EVM）链上执行交易支付的最大 Gas Unit 数量。

### Governance policies（业务风控）
业务风控确定了批准或拒绝某项操作所依据的审批规则。审批规则包括自动拒绝、自动通过、审批门槛和定制工作流。

## I
### Initiator address（发起钱包地址）
发起钱包地址用于支付在 Cobo Portal 上创建智能合约钱包（Safe&#123;Wallet&#125;）的费用。发起钱包地址必须来自 MPC 钱包（机构钱包）。

## M
### Main Account（主账户）
主账户是交易所钱包中的核心账户，全面掌控其关联子账户内的所有资产和操作。

### Main Key（主控私钥）
Multi-Party Computation（MPC）系统中的一种私钥类型，其私钥分片持有者负责签署交易以及创建和管理其他私钥，以维护系统的管理和操作完整性。

### Mobile signers（移动端签名人）
移动端签名人是您团队中的一个成员，该成员使用 [Cobo Guard](/cn/guard/introduction) 来签署交易。

### MPC Wallets（MPC 钱包）
一种利用多方计算（MPC）技术保护数字资产安全的钱包类型。Cobo 提供两种 MPC 钱包：机构钱包和终端用户钱包。

### Multi-Party Computation（MPC）
一种区块链密钥管理的密码学技术。通过 MPC，私钥分片在独立的安全环境中生成、加密，并分布在多方之间。这些参与方在不暴露各自私钥分片或生成完整私钥的情况下，协同完成交易签名。

### Multi-sig transaction（多签交易）
多签交易需要多个私钥持有人在交易执行前共同授权，而不是仅依赖于单个私钥的签名。

## O
### Off-chain transaction policy（链下交易风控规则）
Cobo Portal 的链下交易风控由后台管理，支持您灵活设置交易处理和审批规则。这些规则可以覆盖交易限额、频率、白名单等多个方面。

### On-chain transaction policy（链上交易风控规则）
链上交易风控规则由区块链网络上的智能合约管理。通过这些规则，您可以定义交易的处理流程和自动审批方式。请注意，您必须先设置链上交易规则，才能使用智能合约钱包（Safe&#123;Wallet&#125;）进行提现或与 dApp 交互。

### Organization（团队）
Cobo Portal 中的最高级实体。每个团队在 Cobo Portal 生态系统中独立管理其钱包、风控策略和运营等。

### Organization-Controlled Wallets（机构钱包）
Cobo Portal 上的一种 MPC 钱包类型，其数字资产的所有权和管理权由团队自行掌控。

## R
### Recovery Key（恢复私钥）
Multi-Party Computation（MPC）系统中的一种私钥类型，其私钥分片持有者负责在私钥分片丢失或被破坏的情况下恢复对钱包或系统的访问。

### Risk Control（风险控制）
一套全面的机制，旨在解决数字资产存储、管理和转移中的挑战，包括链上和链下交易风控规则、业务风控规则以及用户角色与权限。

## S
### Safe&#123;Wallet&#125;
Safe&#123;Wallet&#125; 是一款多签钱包和平台，用于在以太坊和流行的 EVM 链上为您存储数字资产。

### Server signers（服务端签名人）
服务端签名人是一个服务器节点，储存了您的 MPC 钱包的私钥分片。服务器签名人使用 TSS Node 软件包来自动签署 MPC 钱包所发起的交易。

### Signer address（签名人地址）
在智能合约钱包（Safe&#123;Wallet&#125;）内，签名人地址用于发起、签署和执行交易。在 Cobo Portal 上，签名人地址可以来自 MPC 钱包（机构钱包）或 Cobo Portal 外部的外部账户（EOA）。

### Signing Key（签名私钥）
Multi-Party Computation（MPC）系统中的一种私钥类型，其私钥分片持有者负责协作签署交易。

### Single-sig transaction（单签交易）
单签交易只需要一个私钥持有人的签名即可授权交易。

### Smart Contract Wallet（智能合约钱包）
智能合约钱包是一种基于区块链技术构建的去中心化 App（dApp），允许用户通过可编程的智能合约管理数字资产。

### Sub Account（子账户）
子账户是交易所钱包中的从属账户，由其主账户创建和管理。

## T
### Transaction Category（交易分类）
在 Cobo Portal 中用于分类和团队团队内转账的用户定义标签，便于更轻松地识别和管理交易。

### Threshold（阈值）
智能合约钱包（Safe&#123;Wallet&#125;）的阈值定义了在执行交易之前，必须有多少个签名人地址批准该交易。

### Threshold Signature Scheme, TSS（门限签名机制）
MPC 钱包使用的一种数字签名协议，使分布的参与方能够共同签署交易。在 MPC-TSS 设置中，可以实现灵活的门限签名机制（n 个私钥分片中的 m 个）。例如，在 2-2 签名方案中，将独立生成两个私钥分片，每笔交易都需要这两个私钥分片共同签名。

### TSS Node
TSS Node 是 MPC 钱包的一个组件，用于生成和管理私钥分片。它与 Cobo Portal 连接，执行密码学操作，包括生成私钥分片、签署交易和生成新的私钥分片。TSS Node 可以是移动端 App（如 Cobo Guard）或部署了 TSS Node 软件包的服务端节点。

### User-Controlled Wallets, UCW（终端用户钱包）
Cobo Portal 上的一种 MPC 钱包类型，其数字资产的所有权和管理权由终端用户通过持有自己的私钥分片来直接控制。

### Vaults
一个 Vault 中包含共享相同根扩展私钥的 MPC 钱包。

### Web3 Wallets（Web3 钱包）
Web3 钱包是一种全托管钱包，支持代币转账、合约调用和消息签名。

### Webhooks
在 Cobo Portal 中，Webhooks 提供实时通知，例如交易状态更新等事件。
