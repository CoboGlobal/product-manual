如果您正在更换新的 iOS 设备、无法再访问旧设备，或重新安装了 Cobo Guard，则需要在新设备上重置 Cobo Guard 以确保其正常使用。

<Note>为避免潜在风险，我们不建议在设备上卸载并重新安装 Cobo Guard。如果您无意间卸载并重装了 Cobo Guard，也需要执行重置流程。在这种情况下，本文档中的“旧设备”和“新设备”均指同一台设备。</Note>

## 适用于已创建私钥分片的 Cobo Guard 用户

### 私钥分片持有者的重置指南

如果您的 Cobo Guard 作为私钥分片持有者参与 MPC 钱包密钥管理，请在执行重置前参考下方指南。

<Info>提示：该流程图并非实际 UI 流程。实际 UI 会根据您选择的设备与备份情况，引导您执行相应步骤（详见下一部分**私钥分片持有者的重置步骤**），但可能不会覆盖某些边缘情况。此流程图提供了所有可能的私钥分片恢复路径，供您参考。</Info>

<img src="/cn/images/cobo-guard/reset&recovery.svg" className="screenshot_diagram" alt="私钥分片持有者重置指南" />

### 私钥分片持有者的重置步骤

1. 在新设备上[下载 Cobo Guard](/cn/guard/download)。

2. 在网页端进入 [Cobo Accounts](https://accounts.cobo.com/)，找到 **MFA Management** 部分，点击 Cobo Guard 重置提示上的 **Complete**。

<img src="/en/images/cobo-guard/notice.png" className="screenshot_modal" alt="重置提示" />

3. 系统会引导您依次进行多个选择（从下图所示的界面开始），以了解您的设备与私钥分片备份情况，并根据您的实际情况展示相应的操作步骤。

<img src="/en/images/cobo-guard/options.png" className="screenshot_modal" alt="选项" />

请根据您适用的情景，查看相应的说明。

#### 情景 1：旧设备可用

如果旧设备上的 Cobo Guard 仍可使用，系统会提示您完成以下事项：

<img src="/en/images/cobo-guard/old-device-available.png" className="screenshot_modal" alt="旧设备可用情形" />

<Note>对于步骤 3 恢复私钥分片，详情请参阅本文档的[**参考：私钥分片恢复步骤**](#steps-to-restore-key-share)部分。</Note>

#### 情景 2：旧设备不可用 + 有有效备份

如果旧设备上的 Cobo Guard 无法使用，但您已进行有效备份且记得备份使用的 iCloud 账号与恢复口令，系统会提示您完成以下事项：

<img src="/en/images/cobo-guard/restore-backup.png" className="screenshot_modal" alt="从备份恢复" />

<Note>对于步骤 3“恢复私钥分片”，请参照[**参考：私钥分片恢复步骤**](#steps-to-restore-key-share)章节中的说明。</Note>

完成所有项目后，点击 **Link New Device**（绑定新设备）。有关绑定新设备的详细流程，详情请参阅本文档的[**参考：绑定新设备步骤**](#steps-to-link-new-device)部分。

#### 情景 3：旧设备不可用 + 无有效备份

系统将提示您[联系客服](mailto:help@cobo.com)以获得人工协助。

## 适用于尚未创建私钥分片的 Cobo Guard 用户

1. 进入 [Cobo Accounts](https://accounts.cobo.com/) 的 **MFA Management** 页面，点击 Cobo Guard 旁的 **Reset**（重置），开始重置流程。系统将引导您进入 **Link New Device**（绑定新设备）对话框。

<img src="/en/images/cobo-guard/link-new-device.png" className="screenshot_modal" alt="绑定新设备" />

2. 在新设备上[下载 Cobo Guard](/en/guard/download)。

3. 在新设备上打开 Cobo Guard，选择 **绑定您的账户**。有关绑定新设备的详细步骤，请参阅[**参考：绑定新设备步骤**](#steps-to-link-new-device)。

---

## 参考：私钥分片恢复步骤 <a id="steps-to-restore-key-share"></a>

<Note>仅当 UI 流程提示您恢复私钥分片时，再执行以下步骤。请勿直接单独阅读此部分——请确保您已完成前序流程，并确实进入了恢复私钥分片的步骤。</Note>

1. 在新设备上打开 Cobo Guard，选择 **Restore existing key shares first**（先恢复现有私钥分片）。Cobo Guard 将开始检查您的私钥分片文件。

2. 选择需要恢复的私钥分片对应的 TSS Node ID，然后点击 **恢复**。

    <img src="/en/images/cobo-guard/common/restore2.png" className="screenshot_modal" alt="TSS Node ID 列表" />

3. 输入备份私钥分片时设置的恢复口令，并点击 **确定**。

    <img src="/en/images/cobo-guard/common/restore3.png" className="screenshot_modal" alt="输入恢复口令界面" />

4. 恢复完成后，Cobo Guard 将显示 **私钥分片恢复成功**。

您可以通过 <img src="/en/images/cobo-guard/common/icon_menu.svg" className="icon" alt="菜单图标" /> → **我的公钥** 检查 TSS Node ID 是否与 Cobo Accounts 显示的一致。

## 参考：绑定新设备步骤 <a id="steps-to-link-new-device"></a>

<Note>仅当 UI 流程提示您绑定新设备时，再执行以下步骤。请勿直接单独阅读此部分——请确保您已完成前序流程，并确实进入了绑定新设备的步骤。</Note>

1. 使用新设备上的 Cobo Guard 扫描 Cobo Accounts 页面弹窗中的二维码。

2. 绑定成功后，[Cobo Accounts](https://accounts.cobo.com/) 页面将显示 **Security Verification**（安全验证）对话框。

<img src="/en/images/cobo-guard/security-verification.png" className="screenshot_modal" alt="安全验证" />

如果您可以使用旧设备，请在旧设备上打开 Cobo Guard 并点击 **Approve**（批准）。  
如果旧设备不可用，请点击 **Cobo Guard not available?**（Cobo Guard 不可用），并选择其他 MFA 方式完成身份验证。

<Note>如果您未设置其他 MFA 方式，请联系[技术支持](mailto:help@cobo.com)。</Note>

验证成功后，系统会将您重定向至 **MFA Management** 页面。在新设备上打开 Cobo Guard，点击 <img src="/en/images/cobo-guard/common/icon_menu.svg" className="icon" alt="菜单图标" /> → **My Accounts**，您将看到与 Cobo Accounts 关联的邮箱地址，表示绑定成功。

<Note>如果您在上述步骤中使用了除 Cobo Guard 以外的 MFA 方式进行验证，您的 Cobo Accounts 将被暂时冻结。需要由团队中的任意[管理员](/cn/portal/organization/roles-and-permissions)解除冻结。如果您是团队中唯一成员，则整个团队将被冻结 24 小时，之后您即可重新登录 Cobo Portal。</Note>

至此，您已完成 Cobo Guard 的重置流程。
