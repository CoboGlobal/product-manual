## 设置账户和团队
作为团队的第一位管理员，请完成以下初始步骤来配置您的账户和团队：

### 创建账户
使用我们销售团队提供的激活链接创建您的 Cobo Portal 账户。

### 设置 Cobo Guard
   1. 登录 Cobo Portal。
      * 如需测试，请使用[开发环境](https://portal.dev.cobo.com/login)。
      * 如需正式使用，请使用[正式环境](https://portal.cobo.com/login)。
   2. 点击左下角的**指南**，然后点击**绑定 Cobo Guard**。
      <img src="/cn/images/get-started/cobo-guard.png" className="screenshot_nav_bar" alt="绑定 Cobo Guard"/>
   3. 按照屏幕上的说明在您的 iOS 设备上安装 Cobo Guard，并将您的账户与 Cobo Guard 关联。

### 配置业务风控规则和用户角色
<Note>如果您正处于测试阶段，想要先快速验证充提币功能，可以暂时跳过此部分，稍后再进行配置。</Note>

<Accordion title="配置业务风控规则和用户角色">
配置业务风控规则和用户角色对于管理团队的访问控制和安全性至关重要。

- [业务风控规则](/cn/portal/organization/governance-intro)定义了关键操作（如删除成员、冻结团队和管理 API Key 等）的审批规则，使用自动通过、自动拒绝和审批门槛等机制。如果默认规则不符合您团队的需求，您可以[编辑业务风控规则](/cn/portal/organization/governance-edit)。

- [用户角色](/cn/portal/organization/roles-and-permissions)使您能够为团队中的成员分配特定权限。当您[为成员分配角色](/cn/portal/organization/assign-roles-to-members)时，他们只能执行该角色允许的操作。例如，具有观察员角色的成员可以查看钱包信息，但不能提币。

举例来说，您可以设置业务风控规则，要求至少两名具有管理员或经理角色的成员批准才能删除成员，如下图所示。

<img src="/cn/images/get-started/delete-members.png" className="screenshot_full_screen" alt="删除成员"/>

</Accordion>

### 邀请成员
   1. 登录 Cobo Portal。
      * 如需测试，请使用[开发环境](https://portal.dev.cobo.com/login)。
      * 如需正式使用，请使用[正式环境](https://portal.cobo.com/login)。
   2. 点击左下角的个人资料图标，然后点击**团队**。
   3. 点击**成员**。在**成员**页面，点击**邀请成员**。
   4. 输入成员的邮箱地址和姓名，然后选择适当的[用户角色](/cn/portal/organization/roles-and-permissions)。
   <Note>请在 Cobo Guard 上确认此邀请是由您发起的。另外，根据默认的业务风控规则，至少需要半数以上的团队管理员在 Cobo Guard 上批准邀请请求。</Note>

   收到邀请后，新成员应按照前面的章节所述创建自己的账户并设置 Cobo Guard。

### 添加链

查看和管理团队可用的链：

   1. 登录 Cobo Portal。
      * 如需测试，请使用[开发环境](https://portal.dev.cobo.com/login)。
      * 如需正式使用，请使用[正式环境](https://portal.cobo.com/login)。
   2. 点击左下角的个人资料图标，然后点击**套餐**。
   3. 导航至**可用公链数**并点击**管理**图标。
   <img src="/cn/images/get-started/add-chain.png" className="screenshot_full_screen" alt="添加链"/>
   4. 在**管理**页面，查看可用链列表。如果您需要的链不在列表中，点击**添加公链**为您的团队添加更多公链。

   <Note>添加链后，该链支持的所有代币都可以在您的钱包中进行充提币操作。</Note>

   <Info>要查看 Cobo 支持的完整代币和链列表，请参阅[支持的代币和链](/cn/portal/supported-tokens-and-chains)。</Info>
